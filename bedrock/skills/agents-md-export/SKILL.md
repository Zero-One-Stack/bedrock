# agents-md-export — mirror the constitution into AGENTS.md

`CLAUDE.md` is Claude-specific. **`AGENTS.md`** is the open "one file, every tool" standard
(Linux Foundation / Agentic AI Foundation; ~60k repos as of 2026) that Cursor, GitHub Copilot,
OpenAI Codex, Aider, Windsurf, Zed, and Gemini CLI read natively. **Claude Code does not yet
read AGENTS.md natively** — but generating it costs nothing and makes the same standard
authoritative across every AI tool that touches the repo.

## Principle

`AGENTS.md` is **short and behavioral** — every line changes what an agent does. It is a
**distillation + pointer**, not a copy of all 25+ rules. The kit's `rules/` remain the source
of truth; AGENTS.md is the cross-tool index.

## Steps

1. Read the project's `.claude/CLAUDE.md` (hard bans, routing table, stack, quality bar) and
   the `.claude/rules/README.md` index.
2. Write `./AGENTS.md` at the project root using the template below.
3. Keep it **under one page** (~120 lines, ~600 words). Add the top comment so editors know
   it's generated.
4. If `AGENTS.md` already exists and was hand-edited (no `<!-- generated -->` marker),
   reconcile rather than clobber: keep the existing file, append a `<!-- bedrock-managed -->`
   block at the end (or warn and stop).

## The template

Use this verbatim, substituting the project's actual stack and engine choice. The template
is engine-agnostic (`styling-engine.md`) and avoids any claim that was removed in the
2026 pivots (no Chakra/Tailwind/CSS-in-JS bans; styling is the project's choice).

```md
<!-- generated by /bedrock:agents-md-export from .claude/CLAUDE.md + .claude/rules/.
     DO NOT edit by hand — edit the source files, then re-run the export. -->

# AGENTS.md

Engineering standard for this repo. **The source of truth is `.claude/CLAUDE.md` +
`.claude/rules/`.** This file is the cross-tool index so Cursor / GitHub Copilot / OpenAI
Codex / Aider / Windsurf / Zed / Claude Code all enforce the same rules. Any AI tool working
in this repo MUST follow it.

## Before writing code (Step 0 Recon — non-negotiable)

Read first:
- `.claude/CLAUDE.md` (hard bans + routing table)
- `.claude/rules/project-specifics.md` (this project's living memory + approved overrides)
- The relevant `.claude/rules/<topic>.md` for the task at hand (see routing below)
- `package.json` scripts, `tsconfig.json` paths, the actual token names in `tokens/` —
  every name in the kit's docs is **illustrative**; confirm before using.

If the request conflicts with the constitution, **name the conflict, cite the rule, and
either refuse the violating part with a compliant alternative or proceed only on an explicit
written override recorded in `project-specifics.md`**.

## Hard bans (any one fails review)

- ❌ **Effector / Redux for server state.** Server state = React Query; local state = React
  primitives. (`rules/services-and-data.md`)
- ❌ **`any`** and **`as` casts** without a runtime guard. (`rules/typescript-and-quality.md`)
- ❌ **Pinned versions** in code/docs; installing non-latest without a written reason.
- ❌ **Hardcoded user-facing strings** — use i18next. (`rules/i18n.md`)
- ❌ **Violating the FSD import rule** (`rules/feature-sliced-design.md`): upward imports,
  same-layer slice imports (except `@x` on `entities/`), deep imports past a slice's
  `index.ts` public API. Barrels that `export *`.
- ❌ **Misplaced FSD code:** business terminology in `shared/`; mutations or action buttons in
  an `entity/`; full-screen `feature/`; layout styling or business logic in `app/`/route
  files; segments named by essence (`components/hooks/utils/modals`); the deprecated
  `processes/` layer; speculative "insignificant" slices.
- ❌ **Server/client runtime leaks** (`rules/services-and-data.md`): entity `*.queries.ts`
  missing `import 'server-only';`; feature `*.action.ts` missing `'use server';`; `'use
  client'` at the top of `app/**/page.tsx` or `src/pages/<route>/ui/<X>Page.tsx`; `@x` on
  `features/widgets/pages` (entities-only).
- ❌ **Editing another team's owned area without their review** (multi-team repos —
  CODEOWNERS + `scope:team-*` tags). (`rules/team-ownership.md`)
- ❌ **Circular dependencies** (transitive cycles + barrel re-export loops).
- ❌ **Testing implementation details** (internal state, private fns, class names).
- ❌ **Shipping a feature without BOTH test layers** — unit/integration AND ≥1 E2E user-flow
  in Playwright. (`rules/testing.md`)
- ❌ **Inaccessible UI** (WCAG 2.2 AA). (`rules/accessibility.md`)
- ❌ **Desktop-first / hardcoded pixel breakpoints**; touch targets under 44px.
- ❌ **Security holes** (XSS, secrets in client / `NEXT_PUBLIC_*`, unscheme-checked
  `href`/`src`, tokens in `localStorage`, unvetted deps). (`rules/security.md`)

## Stack (verify against `project-specifics.md`)

Next.js (App Router) · React · TypeScript strict · **Feature-Sliced Design** (root `app/`
routing + FSD layers under `src/`, enforced by Steiger) · styling engine of the project's
choice (recorded in `project-specifics.md`; 3-tier DTCG design tokens recommended) · TanStack
React Query · React Hook Form + Zod · i18next · Testing Library + MSW · Storybook (CSF3 +
autodocs + play + a11y addon + visual regression).

## How to build (the working order)

1. **Plan first** for anything multi-component: decompose by FSD layer
   (`rules/architecture.md`).
2. **Place** each unit on the right layer (`rules/feature-sliced-design.md`); never
   speculatively extract a slice (the insignificant-slice trap).
3. **Build bottom-up:** `shared` → `entities` → `features` → `widgets` → `pages` → route
   wiring. Reuse before creating (tokens, atoms, mocks, hooks, the class-merge util).
4. **Verify** with `/verify-build` (Steiger + dependency-cruiser + lint + tests + tokens
   resolving).
5. **Review** against the constitution (FSD placement, a11y, tokens, tests).

## Enforcement layers (what catches what)

This kit's enforcement is **layered**, not all-or-nothing. PreToolUse hooks fire at write
time on Edit/Write/MultiEdit; ESLint backstops existing code and refactors; Steiger /
dependency-cruiser / OPA run in CI on the whole tree; the `frontend-reviewer` agent reads
the diff against the constitution. Some hook bypasses exist (pipe mode, MCP subagents,
shell-issued writes) — see `rules/governance.md`'s enforcement matrix for the full picture.

## Quality bar

TS strict / no `any` · coverage ≥ 80% (90%+ new code) · unit + E2E both present ·
accessible by default · i18n everything · secure (no XSS/secrets/unvetted deps) ·
tokens for every design value (when the project uses tokens) · lint/typecheck/format/a11y
clean before "done."

---

**Edit `.claude/CLAUDE.md` and `.claude/rules/`, then re-run `/bedrock:agents-md-export` to
regenerate this file. Do not edit AGENTS.md directly.**
```

## When AGENTS.md already exists

The skill detects this two ways:

1. **Generated by this skill** — the file starts with `<!-- generated by
   /bedrock:agents-md-export …`. Safe to overwrite — just regenerate.
2. **Hand-edited or third-party** — no `<!-- generated -->` marker. **Do not clobber.** Instead:
   - Print a one-line warning.
   - Write the kit's distillation to `./AGENTS.bedrock.md` instead.
   - Ask the user to merge manually or rename.

## Done when

`./AGENTS.md` exists at the project root, under one page, with the routing table pointing at
`.claude/rules/<topic>.md`. Report what was written and confirm the kit's `rules/` folder
remains the source of truth. The skill runs automatically as part of `/bedrock:kit-init`; re-run
any time the constitution changes (or wire it into a hook that re-runs on `CLAUDE.md` edits —
optional).
